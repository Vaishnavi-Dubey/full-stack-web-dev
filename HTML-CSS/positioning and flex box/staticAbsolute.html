CSS Positioning: Static vs Absolute

Static Positioning (default):
This box is positioned statically, which means it flows normally within the document. It respects margins, paddings, and appears in the order it is written in HTML. The top, left, right, and bottom properties have no effect here.
Absolute Positioning:
This box is positioned absolutely relative to the nearest ancestor with positioning (here, the container). It is removed from the normal document flow, so it overlaps other content without affecting their position. Using top: 30px and left: 250px, it is placed exactly within the container.

Key Concepts Explained:

  • Static Positioning: Default for all HTML elements; element appears in normal flow; positioning properties like top, left have no effect.
  • Absolute Positioning: Element is removed from document flow; positioned relative to nearest ancestor with position: relative|absolute|fixed.
  • Positioning Context: An ancestor with position other than static creates a positioning context. Without it, absolutely positioned elements are positioned relative to the document body.
  • Effect on Layout: Static elements take up space and push other elements; absolute elements overlap others and take no space.
  • Stacking: Absolutely positioned elements can overlap static elements; use z-index to control layering.
